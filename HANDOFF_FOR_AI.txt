-----------------

IP: 172.0.0.1

Current startup behavior:
On every boot the firmware clears saved external Wi-Fi credentials, disconnects
from old Wi-Fi, and starts the DONGL access point. This was added because the
dongle previously remembered an external network and stopped showing DONGL.

Local firmware endpoints:

http://31.76.20.227

Cloud macro policy:
Allowed and blocked cloud macro policy is now centralized in:
macro_policy.json

Server reads it at startup from:
/opt/macro-controller/macro_policy.json

Local server default path:
server/app.py reads ../macro_policy.json

Firmware build generates this header from macro_policy.json:
firmware/src/macro_policy_generated.h

Generator script:
scripts/generate_macro_policy.py

PlatformIO runs the generator before build through firmware/platformio.ini.

Current allowed cloud macro families:
- DELAY
- GUI r
- STRING text, length-limited and blocked-word checked
- ENTER
- TAB, SHIFT TAB, ESC, BACKSPACE, DELETE
- UP, DOWN, LEFT, RIGHT
- F1-F12
- CTRL A/C/F/S/V/X/Z
- ALT TAB

Blocked words inside STRING are also in macro_policy.json.

Notes and caveats
-----------------
The current source tree may have an uncommitted README.md change made outside
the latest firmware change. Do not overwrite user changes without checking.

If the cloud macro is queued but does not run:
1. Check http://31.76.20.227/api/dongles
2. Check whether the dongle is online.
3. Remember that after the latest firmware change the dongle resets saved Wi-Fi
   on each boot, so cloud mode only works after connecting it to external Wi-Fi
   during that current power session.

Security boundary
-----------------
Do not implement arbitrary Ducky Script execution through the VPS/cloud channel.
The cloud path may queue user-written macro text only if it stays limited to
the strict allowlist above and is validated before queueing and again on-device.
